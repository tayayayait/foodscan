# 한국형 식품 스캔 분석 서비스 UI/UX 상세서

## 1. 문서 목적

이 문서는 한국형 식품 스캔 분석 서비스의 MVP 구현 기준 문서다. 개발자와 디자이너는 본 문서를 기준으로 화면, 컴포넌트, 상태값, 접근성, 예외 처리를 동일하게 구현한다.

## 2. 제품 정의

### 2.1 서비스 목표

사용자가 바코드, 제품명, 제품 사진을 입력하면 식품 기본정보, 영양성분, 원재료, 첨가물, 알레르기, 회수·판매중지 이력, 대체상품을 모바일 중심 UI에서 확인할 수 있게 한다.

### 2.2 MVP 범위

MVP는 완전 자동 식별 앱이 아니다. GS1 상품DB/API를 제외하고 다음 소스를 단계적으로 조회하는 식품 분석 앱으로 정의한다.

1. 자체 검수 DB
2. 식약처·공공 API
3. Open Food Facts
4. Gemini OCR/Search
5. 임시 제품 생성
6. 사용자 확인 또는 관리자 검수
7. 자체 DB 저장

### 2.3 핵심 사용자

| 사용자           | 목적                                                         | 주요 화면                                      |
| ---------------- | ------------------------------------------------------------ | ---------------------------------------------- |
| 일반 사용자      | 바코드·사진·제품명으로 식품 안전성과 영양 정보를 빠르게 확인 | 홈, 스캔, 검색 결과, 제품 결과, 개인 기준 설정 |
| 식이 제한 사용자 | 특정 성분 포함 여부와 식이 선호 적합성을 즉시 확인           | 제품 결과, 알레르기 탭, 개인 기준              |
| 보호자           | 일반 점수와 상세 근거를 함께 확인                            | 홈, 제품 결과, 대체상품                        |
| 관리자           | 미식별 또는 임시 제품을 검수하고 공식 DB화                   | 관리자 대시보드, 검수 상세                     |

## 3. 정보 신뢰도 원칙

### 3.1 데이터 출처 우선순위

| 우선순위 | 출처              | UI 표시명      | 신뢰도 배지      | 사용 조건                                               |
| -------: | ----------------- | -------------- | ---------------- | ------------------------------------------------------- |
|        1 | 자체 검수 DB      | 검수 완료      | `verified`       | 관리자가 승인한 제품                                    |
|        2 | 식약처·공공 API   | 공공데이터     | `public_api`     | 제품명, 제조사, 품목보고번호, 원재료, 회수 이력 확인 시 |
|        3 | Open Food Facts   | 사용자 공개 DB | `open_db`        | 국내 공공 API 미매칭 또는 글로벌 제품 보조              |
|        4 | Gemini OCR/Search | AI 추정        | `ai_estimated`   | 이미지 OCR, 검색 구조화, 쉬운 설명 생성                 |
|        5 | 사용자 입력       | 사용자 제보    | `user_submitted` | 미식별 제품 등록 시                                     |

### 3.2 AI 결과 표시 규칙

Gemini 결과는 공식 데이터 대체가 아니다. AI가 생성하거나 구조화한 값은 다음 중 하나를 반드시 표시한다.

| 상태             | 표시 문구        | 사용 조건                      |
| ---------------- | ---------------- | ------------------------------ |
| `ai_estimated`   | AI가 추정한 정보 | OCR 또는 검색 기반 구조화 결과 |
| `needs_review`   | 검수 필요        | 관리자 승인 전 임시 제품       |
| `low_confidence` | 정확도 낮음      | confidence < 0.7               |

### 3.3 confidence 기준

|          confidence | UI 처리                                            | 저장 상태                     |
| ------------------: | -------------------------------------------------- | ----------------------------- |
|           0.90 이상 | 자동 표시 가능. 출처 배지 표시                     | `provisional` 또는 `verified` |
| 0.70 이상 0.90 미만 | 사용자 확인 배너 표시                              | `provisional`                 |
| 0.50 이상 0.70 미만 | 관리자 검수 우선 큐로 전송                         | `needs_review`                |
|           0.50 미만 | 제품 생성 전 사용자에게 재촬영 또는 직접 입력 요청 | `rejected_or_retry`           |

## 4. 전체 사용자 흐름

### 4.1 바코드 입력 흐름

```text
홈
→ 바코드 스캔 버튼 탭
→ 카메라 권한 확인
→ 바코드 인식
→ 자체 DB 조회
→ 식품안전나라 C005 조회
→ I2570 보조 조회
→ Open Food Facts 조회
→ 매칭 성공 시 제품 결과
→ 매칭 실패 시 사진 촬영 또는 제품명 검색 유도
```

### 4.2 제품명 검색 흐름

```text
홈 검색창
→ 검색어 입력
→ 자체 DB 검색
→ 식약처 I1250 검색
→ Open Food Facts 검색
→ 결과 목록
→ 제품 선택
→ 제품 결과
```

### 4.3 사진/OCR 흐름

```text
홈
→ 사진 스캔 버튼 탭
→ 전면 사진 또는 영양성분표 촬영
→ Gemini OCR
→ 제품명/브랜드/용량/영양성분/원재료 추출
→ 후보 제품 JSON 생성
→ 사용자 확인
→ 임시 제품 생성
→ 관리자 검수 큐 저장
```

### 4.4 관리자 검수 흐름

```text
관리자 로그인
→ 미검수 제품 목록
→ 검수 상세
→ OCR 원문 확인
→ Gemini 구조화 결과 확인
→ 공공 API 매칭 결과 확인
→ 승인 / 수정 / 반려
→ 자체 DB 저장 또는 반려 사유 기록
```

| 자동완성    | 최근 검색어 5개, 제품 후보 5개까지      |

### 8.3 카드

| 속성      | 값                  |
| --------- | ------------------- |
| 배경      | `#FFFFFF`           |
| 라운드    | 8px                 |
| 보더      | `1px solid #E5E7EB` |
| 그림자    | `shadow.card`       |
| 내부 패딩 | 16px                |
| 카드 간격 | 12px                |

카드 내부에 다른 카드를 중첩하지 않는다. 반복 항목, 제품 요약, 대체상품, 검수 항목에만 카드 스타일을 사용한다.

### 8.4 배지

| 종류      | 높이 | 패딩 |       폰트 | 라운드 |
| --------- | ---: | ---: | ---------: | -----: |
| 상태 배지 | 24px |  8px | 12px / 600 |    4px |
| 출처 배지 | 22px |  6px | 12px / 500 |    4px |
| 위험 배지 | 26px |  8px | 13px / 700 |    4px |

배지는 색상만으로 의미를 전달하지 않는다. 예: `회수 이력 있음`, `AI 추정`, `검수 완료`.

### 8.5 칩

| 속성        | 값         |
| ----------- | ---------- |
| 높이        | 36px       |
| 패딩        | 12px       |
| 라운드      | 999px      |
| 기본 배경   | `#F1F5F9`  |
| 선택 배경   | `#CCFBF1`  |
| 선택 텍스트 | `#115E59`  |
| 폰트        | 14px / 600 |

개인 기준은 칩 대신 토글 목록으로 제공한다.

```text
성분 알림: 글루텐, 유당, 아황산류, 대두, 팜유
식이 선호: 채식, 비건, 돼지고기 제외
```

### 8.6 탭

| 속성          | 값                  |
| ------------- | ------------------- |
| 높이          | 44px                |
| 배경          | `#F1F5F9`           |
| 활성 배경     | `#FFFFFF`           |
| 활성 텍스트   | `#0F766E`           |
| 비활성 텍스트 | `#475569`           |
| 보더          | `1px solid #E5E7EB` |
| 라운드        | 8px                 |

탭은 한 줄에 들어가지 않으면 가로 스크롤을 허용한다. 탭 버튼 각각은 최소 88px이다.

### 8.7 모달·바텀시트

모바일은 바텀시트, 태블릿 이상은 중앙 모달을 기본으로 사용한다.

| 속성       |          모바일 바텀시트 |     데스크톱 모달 |
| ---------- | -----------------------: | ----------------: |
| 너비       |                     100% |             560px |
| 최대 높이  |                     88vh |              80vh |
| 라운드     |                 상단 8px |               8px |
| 배경       |                `#FFFFFF` |         `#FFFFFF` |
| overlay    | `rgba(15, 23, 42, 0.45)` |              동일 |
| padding    |                     20px |              24px |
| close 버튼 |        우측 상단 44x44px | 우측 상단 44x44px |

닫기 방식:

1. 닫기 버튼
2. Escape
3. overlay 클릭. 단 저장되지 않은 변경이 있으면 확인 모달 표시

### 8.8 토스트

| 종류 | 색상   | 위치               |             지속 시간 |
| ---- | ------ | ------------------ | --------------------: |
| 성공 | good   | 모바일 하단 nav 위 |                3000ms |
| 오류 | danger | 모바일 하단 nav 위 | 수동 닫기 또는 6000ms |
| 정보 | info   | 모바일 하단 nav 위 |                4000ms |

토스트는 핵심 오류 설명을 대체하지 않는다. 입력 오류는 해당 필드 하단에 표시한다.

### 8.9 스켈레톤

| 위치             | 규격                   |
| ---------------- | ---------------------- |
| 제품 카드 이미지 | 64x64px                |
| 제목 라인        | width 70%, height 18px |
| 보조 라인        | width 45%, height 14px |
| 그래프 바        | height 12px            |

스켈레톤 애니메이션은 1200ms shimmer를 사용한다. `prefers-reduced-motion`이 활성화되면 애니메이션을 정적 배경으로 대체한다.

| 원재료 출처     | C002, OCR, 사용자 입력                               |
| 영양성분 출처   | 공공데이터, Open Food Facts, OCR                     |
| 회수 출처       | I0490                                                |
| AI 생성 여부    | Gemini 사용 항목 명시                                |
| 마지막 업데이트 | `YYYY.MM.DD HH:mm`                                   |

## 9.5 개인 기준 설정 화면

### 9.5.1 목적

Yuka식 Food Preferences 구조처럼 사용자별 성분 알림과 식이 선호를 설정한다. 개인 기준은 제품 점수를 바꾸지 않고 별도 개인 경고와 대체상품 추천 필터에만 반영한다.

### 9.5.2 구조

```text
AppBar
현재 개인 기준 요약
성분 알림
식이 선호
저장 버튼
```

### 9.5.3 성분 알림

| 항목     | 적용 방식                   |
| -------- | --------------------------- |
| 글루텐   | 밀·글루텐 포함 시 경고      |
| 유당     | 우유·유당 성분 포함 시 경고 |

| ------------- | ------------------------------ |

### 9.7.5 검수 상세

검수 상세는 좌측 원문, 우측 편집 폼 구조를 사용한다.

| 영역               | 내용                                             |
| ------------------ | ------------------------------------------------ |
| OCR 원문           | 제품 전면, 영양성분표, 원재료 OCR 텍스트         |
| Gemini 구조화 결과 | JSON viewer, 필드별 confidence                   |
| 공공 API 매칭 결과 | C005, I2570, I1250, C002, I0490 후보             |
| 편집 폼            | 제품명, 제조사, 식품유형, 용량, 영양성분, 원재료 |
| 액션               | 승인, 수정 저장, 반려                            |

승인 버튼 활성 조건:

1. 제품명 존재
2. 제조사 또는 출처 중 하나 존재
3. 바코드가 있으면 8-14자리 숫자
4. 회수 이력 조회 완료 또는 조회 실패 사유 기록
5. 원재료 또는 영양성분 중 하나 이상 존재

## 12. API 연동 UI 기준

### 12.1 조회 단계 표시

검색 또는 스캔 후 1.5초 이상 걸리면 단계형 로딩을 표시한다.

```text
제품 정보를 확인 중입니다
1. 자체 DB 확인
2. 공공데이터 조회
3. 공개 DB 조회
4. AI 보강
```

완료된 단계는 체크 아이콘, 진행 중 단계는 spinner, 실패 단계는 `건너뜀` 라벨을 표시한다.

### 12.2 API 실패 처리

| 실패 지점            | 사용자 메시지                       | 다음 행동                |
| -------------------- | ----------------------------------- | ------------------------ |
| 자체 DB 실패         | 표시하지 않음                       | 공공 API 계속 조회       |
| 공공 API 실패        | 공공데이터 조회가 지연되고 있습니다 | Open Food Facts 조회     |
| Open Food Facts 실패 | 공개 DB 조회가 지연되고 있습니다    | Gemini 또는 직접 입력    |
| Gemini 실패          | 사진 정보를 읽지 못했습니다         | 재촬영, 직접 입력        |
| 전체 실패            | 제품을 찾지 못했습니다              | 사진으로 등록, 직접 입력 |

## 14. 예외 처리 기준

### 14.1 제품 미식별

조건:

```text
자체 DB, 공공 API, Open Food Facts에서 모두 매칭 실패
```

UI:

```text
제목: 제품을 찾지 못했습니다

CTA 2: 직접 입력
CTA 3: 다시 검색
```

### 14.2 후보 다중 매칭

조건:

```text
제품명 유사도 높은 후보가 2개 이상이고 confidence 차이가 0.15 미만
```

UI:

```text
제목: 어떤 제품인가요?
본문: 비슷한 제품이 여러 개 있습니다
후보 카드 최대 5개
CTA: 이 제품 선택
보조 CTA: 찾는 제품 없음
```

### 14.3 영양성분 없음

UI:

```text
영양성분 정보가 없습니다
영양성분표 사진을 추가하면 분석 정확도가 높아집니다
CTA: 사진 추가
```

점수는 `산출 불가`로 표시한다. 추정 점수를 임의로 표시하지 않는다.

### 14.4 원재료 없음

```text
원재료 정보가 없습니다
원재료명 사진을 추가하거나 직접 입력할 수 있습니다
```

알레르기 경고는 `확인 불가` 상태로 표시한다.

### 14.5 회수 API 실패

회수·판매중지 API 조회 실패는 위험 없음으로 처리하지 않는다.

UI:

```text
회수 이력을 확인하지 못했습니다
네트워크 상태를 확인한 뒤 다시 시도하세요
```

상태 배지는 `회수 확인 실패`로 표시한다.

## 17. QA 체크리스트

### 17.1 모바일

- 360px 폭에서 가로 스크롤이 없어야 한다.
- 검색창, 스캔 버튼, 하단 내비게이션의 터치 영역은 44x44px 이상이어야 한다.
- 제품명 3줄 이상, 제조사 정보 없음, 이미지 없음 상태가 깨지지 않아야 한다.
- 회수 경고와 알레르기 경고가 동시에 표시되어도 첫 화면에서 핵심 문구가 보여야 한다.
- 하단 내비게이션이 CTA 버튼을 가리지 않아야 한다.

### 17.2 데스크톱

- 1024px 이상에서 제품 결과는 메인+보조 2컬럼이어야 한다.
- 관리자 테이블은 컬럼 폭이 유지되어야 하며, 넘치는 경우 테이블 영역만 가로 스크롤한다.
- 상세 Drawer가 열렸을 때 본문 스크롤과 Drawer 스크롤이 분리되어야 한다.

### 17.3 데이터 상태

- 점수 산출 불가 상태에서 임의 점수를 표시하지 않아야 한다.
- 회수 API 실패를 위험 없음으로 표시하지 않아야 한다.
- AI 추정 정보에는 항상 출처 배지가 있어야 한다.
- confidence < 0.7 결과는 사용자 확인 또는 관리자 검수로 보내야 한다.
- 원재료 없음 상태에서는 알레르기 `안전` 표현을 쓰지 않아야 한다.

## 19. 구현 금지 사항

1. GS1 상품DB/API를 MVP 필수 의존성으로 두지 않는다.
2. Gemini 결과를 공식 데이터처럼 표시하지 않는다.
3. 회수 API 실패를 `위험 없음`으로 처리하지 않는다.
4. 원재료 정보가 없는데 알레르기 안전 판정을 내리지 않는다.
5. 점수 산출 불가 상태에서 추정 점수를 임의 표시하지 않는다.
6. 색상만으로 좋음·주의·위험을 구분하지 않는다.
7. 모바일에서 44px 미만 터치 타깃을 만들지 않는다.
8. 카드 안에 카드를 중첩하지 않는다.
