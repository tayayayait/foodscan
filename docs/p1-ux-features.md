## 검색 고도화

- `/search`는 출처, 카테고리, 정렬 필터를 제공한다.
- 검색 결과 카드는 제품명과 브랜드에 현재 검색어 매칭 하이라이트를 적용한다.
- 최근 검색어와 브라우저 로컬 스토리지를 최대 5개까지 보관한다.
- 기존 검색 제품명 후보는 입력창 자동완성 목록으로 표시한다.
- 검색 후보 간 confidence 차이가 `0.15` 미만이면 후보 선택 패널을 표시한다.

## 스캔 입력

- 홈 화면은 바코드 스캔 진입만 제공한다.
- `/scan`은 실시간 바코드 카메라 스캔과 수동 바코드 번호 입력만 제공한다.
- 카메라가 바코드를 인식하면 즉시 제품 조회로 이동하지 않고 번호 입력칸에 값을 채운다. 사용자는 번호를 확인·수정한 뒤 검색 버튼을 눌러 제품 상세 조회로 진행한다.
- 제품 사진을 업로드해 제품을 찾거나 OCR 임시 제품을 생성하는 사진 스캔 흐름은 제공하지 않는다.

## 기록 인사이트

- `/history`는 최근 스캔 기록의 평균 점수, 점수 가능 수, 주의 제품 수, 검수 필요 수를 요약한다.
- 기록 화면은 A-E 등급 분포와 최다 반복 위험 요인을 표시한다.
- 기록 화면은 저장 제품, 최근 스캔 제품, 임시 제품을 합산해 안전 감시 패널을 표시한다.
- 사용자는 전체 기록과 주의 제품만 보는 필터를 전환할 수 있다.
- 주의 제품은 회수 이력, 검수 필요, 개인 기준 경고, 고위험 첨가물, 당류·나트륨·포화지방 과다 기준으로 선별한다.

## 저장 제품

- 제품 상세의 북마크 버튼은 현재 제품을 저장하거나 저장 해제한다.
- 저장 제품은 `localStorage`에 제품 스냅샷과 저장 시각을 함께 보관한다.
- `/history`는 저장 제품만 보는 `저장` 탭을 제공한다.
- 저장 제품은 최근 스캔 목록에 없어도 제품 상세 조회 대상으로 사용한다.

## 신뢰 센터

- `/settings`는 독립 점수, 비후원 추천, 검수 전 표시, 기록 비판매, 의학 판단 제외 원칙을 표시한다.
- 설정 화면은 출처 등급별 점수 사용 범위를 표시한다.
- 설정 화면은 Yuka식 구조에 맞춰 개인 기준 진입점을 포함한다.
- 설정 화면은 최근 스캔, 저장 제품, 임시 제품, 개인 기준, 임시 초안, 최근 검색어의 로컬 저장 상태를 표시한다.
- 설정 화면은 로컬 데이터 JSON 파일 내보내기와 가져오기를 제공한다.
- 데스크톱 사이드바와 모바일 하단 탭은 홈, 스캔, 기록, 설정만 1차 항목으로 제공한다. 개인 기준과 관리자 검수는 설정 화면에서 진입한다.
- `로컬 데이터 모두 삭제`는 위 로컬 저장 항목 전체를 삭제한다.

## 개인 기준

- `/preferences`는 설정 화면의 하위 진입점으로 제공하며 성분 알림과 식이 선호를 저장한다.
- 저장된 개인 기준은 제품 점수를 바꾸지 않고 별도 개인 경고로 표시한다.
- legacy 저장값은 읽을 때 정규화해 지원 가능한 알레르기 값을 성분 알림으로 이관하고, 잘못된 값과 중복 항목은 제거한다.
- 제품 상세 사이드 패널은 현재 선택한 성분 알림과 식이 선호를 요약한다.

## 제품 상세

- 상단 판정 카드는 제품 이미지, 검수 출처, confidence, 원형 점수, 판정 문구를 한 영역에서 보여준다.
- 점수는 영양 60점, 첨가물 30점, 인증 10점 breakdown으로 표시한다.
- 회수/판매중지, 성분 알림, 식이 선호 충돌, 고위험 첨가물, 영양 위험 요인은 `핵심 확인 항목`으로 우선 노출한다.
- 점수 계산이 불가능한 저신뢰 AI/OCR 또는 사용자 제보 제품은 숫자 점수 대신 검수 필요 상태를 표시한다.
- 영양 정보는 제품 상세에서 100g/ml 기준만 표시한다.
- 첨가물 탭은 category별 그룹과 함께 위험도 문구, severity 색상, 근거 문구를 표시한다.
- 미분류 첨가물은 `additive_review_queue` 적재를 시도한다. 브라우저 세션 내 동일 제품/첨가물 조합은 중복 적재하지 않는다.

## 사용자 제보 등록

- `/register`는 제품 사진, 제품명, 바코드, 원재료, 영양성분표 사진, 원재료 사진, 100g 기준 영양값, 알레르기 표시, 첨가물 입력을 제공한다.
- 등록 화면은 제보 품질 점수와 차단/검수 필요 항목을 표시한다.
- 등록 제품의 `confidence`와 `status`는 [사용자 제보 품질 정책](./submission-quality-policy.md)에 따라 계산한다.
- 제품 사진은 사용자 제보 근거로만 사용하며, 사진 스캔 OCR로 제품을 찾는 흐름은 제공하지 않는다.

## 대체상품

- 대체상품 추천은 바코드 조회로 확정된 현재 제품을 기준으로 `searchAlternativeCandidates`가 `nutrition_products`의 같은 소분류·대표식품군 후보를 먼저 조회한다.
- 공공 영양 DB 후보가 없거나 Gemini가 점수형 후보를 고르지 못하면, Gemini가 현재 제품의 영양 부담을 해석해 네이버쇼핑 검색어를 생성한다. 네이버쇼핑 상위 결과는 Gemini가 다시 검토해 실제 영양 대체식품 후보만 통과시킨다.
- `recommendAlternativeProducts`는 후보 목록을 Gemini에 전달해 좁은 제품 형식, 섭취 경험, 구매 의도, 사용 맥락, 영양 개선 가능성을 함께 보고 추천 후보를 직접 랭킹한다.
- Gemini는 후보를 새로 만들거나 영양성분·가격·이미지를 수정하지 않는다. 실제 표시 상품은 공공 영양 DB 등 점수 산출 가능한 후보에 한정한다.
- Gemini가 추천 목록에 포함하고 `fitScore >= 70`을 반환한 후보만 점수 기반 추천 평가에 사용한다. Gemini 호출이 실패하면 넓은 카테고리 후보를 임의로 fallback하지 않고 추천을 표시하지 않는다.
- 네이버쇼핑 검색 결과는 Edge Function에서 `NAVER_SHOPPING_CLIENT_ID`와 `NAVER_SHOPPING_CLIENT_SECRET`으로 조회하고, 점수 후보와 이름이 매칭되는 경우에만 `shoppingOffer`로 붙인다. 상품명 중간에 `찹쌀`, 용량 등 수식어가 삽입된 쇼핑 상품명은 후보명 글자 순서가 보존되면 매칭 대상으로 본다.
- 점수형 추천은 Gemini 추천 순서를 우선 적용한다. 점수형 추천 컨텍스트가 없으면 Gemini 쇼핑 검색 계획 경로로 전환한다.
- 현재 제품, 중복 후보, 회수 이력 후보, 점수 산출 불가 후보, B등급 미만 후보는 추천에서 제외한다.
- 현재 개인 기준의 성분 알림 또는 식이 선호와 충돌하는 후보는 추천에서 제외한다.
- 현재 제품의 점수를 산출할 수 있으면 최소 5점 이상 높고, 당류·나트륨·포화지방·열량·첨가물 중 최소 1개가 낮은 후보만 표시한다.
- 현재 제품의 점수를 산출할 수 없으면 B등급 이상이며 점수 산출 가능한 후보를 표시하고 `점수 산출 가능` 사유를 붙인다.
- 쇼핑형 추천은 기본적으로 Gemini 영양 방향, 네이버쇼핑 가격, 판매처, 링크를 표시한다. Open Food Facts 영양성분 보강이 성공하고 점수 기준을 통과한 경우에만 참고 점수와 등급을 표시한다. 가격은 정렬 기준이 아니라 구매 가능성 표시용이다.
- 추천 카드는 점수형 후보에는 후보 점수, 등급, 개선 점수, 추천 사유, 네이버쇼핑 가격을 표시하고 쇼핑형 후보에는 가격과 영양 대체 방향을 표시한다.

## 제약

- 대체상품 추천 품질은 `nutrition_products` 적재 상태, Gemini의 실제 대체 가능성 판정, 쇼핑 이름 매칭 품질에 의존한다.
- `additive_review_queue`는 현재 public insert 정책을 사용한다.
- 전체 관리자 승인 플로우는 `SUPABASE_SERVICE_ROLE_KEY`와 `ADMIN_ACCESS_CODE`가 설정되어야 정상 동작한다.
