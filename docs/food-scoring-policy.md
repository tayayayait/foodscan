# 식품 점수 정책

마지막 업데이트: 2026-05-13

이 문서는 식품 전용 제품 평가 점수의 구조를 정의한다. Yuka의 공개 식품 평가 구조를 참고하되, 국내 공공데이터와 현재 구현 범위에 맞게 적용한다.

## 원칙

- 식품만 평가한다. 화장품, 의약품, 생활용품은 범위에서 제외한다.
- 점수는 제품 자체의 일반 품질 판단이다. 성분 알림, 식이 선호, 회수 이력은 별도 경고로 분리한다.
- 출처가 불명확한 값은 추정값으로 표시한다. 확정값처럼 표시하지 않는다.
- 확인 가능한 데이터가 부족하면 숫자 점수를 만들지 않는다.
- 공식 인증 또는 검증된 데이터가 없으면 인증 보너스를 부여하지 않는다.

## 점수 구조

| 영역           | 배점 | 처리 방식                                    |
| -------------- | ---: | -------------------------------------------- |
| 영양           |   60 | 100g/ml 기준 영양성분으로 계산               |
| 첨가물         |   30 | 첨가물 위험도별 감점                         |
| 공식 긍정 인증 |   10 | 신뢰 가능한 출처에서 확인된 공식 인증만 가점 |

공식 인증 데이터가 없으면 해당 10점은 `확실한 정보 없음`으로 표시하고 0점 처리한다. 이는 위험 판정이 아니라 미확인 보너스다.

## 등급

|   점수 | 등급 | 사용자 문구      |
| -----: | ---- | ---------------- |
| 85-100 | A    | 매우 좋음        |
|  70-84 | B    | 좋음             |
|  55-69 | C    | 보통             |
|  40-54 | D    | 주의             |
|   0-39 | E    | 피하는 것을 권장 |

점수 계산이 불가능하면 `평가 불가`를 표시한다. 평가 불가는 좋은 제품이라는 의미가 아니다.

## 영양 계산

필수 입력값:

- 열량
- 당류
- 나트륨
- 포화지방

보조 입력값:

- 단백질
- 식이섬유
- 과일/채소/견과류 함량

필수 입력값 중 2개 이상이 없으면 영양 점수를 계산하지 않는다. 1개만 누락된 경우 임시 점수를 만들 수 있으나, 제품 상세에 `영양 정보 일부 누락`을 표시한다.

영양성분 우선 출처는 Supabase `nutrition_products`에 적재한 식품의약품안전처 가공식품 영양 DB다. 바코드 조회로 품목제조보고번호를 확보하면 해당 번호로 먼저 조회하고, 실패 시 제품명 기반 조회와 공공데이터 API fallback을 사용한다.

## 첨가물 계산

첨가물 점수는 `src/lib/additive-dictionary.ts`의 위험도 메타데이터를 사용한다. Supabase에는 같은 기준을 `public.additive_risk_profiles`로 저장해 검수와 향후 업데이트의 기준 테이블로 사용한다.

| riskLevel       | 감점 | 추가 처리      |
| --------------- | ---: | -------------- |
| `risk_free`     |    0 | 없음           |
| `limited_risk`  |    4 | 없음           |
| `moderate_risk` |   10 | 주의 항목 표시 |
| `high_risk`     |   18 | 총점 49점 상한 |
| `unknown`       |    0 | 검수 큐 전송   |

`high_risk` 첨가물이 있으면 첨가물 영역 배점과 무관하게 전체 점수 상한을 49점으로 제한한다.

미분류 첨가물은 점수에 반영하지 않는다. 검수 전까지 추정 감점을 만들지 않고 `확실한 정보 없음`으로 표시한다.

`혼합제제`처럼 구체 첨가물명이 아닌 항목은 `unknown`으로 둔다. `합성착향료`, `합성향료`, `착향료`처럼 향료 계열로 식별되지만 구체 물질과 함량이 없는 항목은 `limited_risk`로 분류한다.

상세 기준은 [첨가물 위험도 정책](./additive-risk-policy.md)을 따른다.

## 공식 긍정 인증

다음 조건을 모두 만족할 때만 최대 10점을 가산한다.

- 인증명이 공식 유기농/친환경 인증 체계와 일치한다.
- 인증 출처가 식약처 공공데이터, 검수 완료 DB, 검증된 공개 DB 중 하나다.
- OCR 또는 제품명 추정만으로 판단하지 않는다.

공식 인증 데이터가 없으면 `확실한 정보 없음`으로 표시한다.

## 별도 경고

다음 항목은 제품 점수와 분리해 최상단 경고로 표시한다.

| 항목                | 표시 방식                    |
| ------------------- | ---------------------------- |
| 회수/판매중지 이력  | 점수보다 위에 빨간 경고 배너 |
| 성분 알림 매칭      | 개인 위험 경고               |
| OCR 저신뢰 결과     | 임시 분석/검수 필요 경고     |
| 검수 전 사용자 제보 | 검수 필요 경고               |
| 식이 선호 충돌      | 개인 기준 경고               |

회수/판매중지 이력이 현재 제품과 명확히 매칭되면 총점과 관계없이 `구매·섭취 전 확인 필요`를 표시한다.

## 점수 결과 구조

```ts
type ScoreResult = {
  score: number | null;
  grade: "A" | "B" | "C" | "D" | "E" | null;
  severity: "good" | "normal" | "caution" | "danger" | "info";
  summary: string;
  breakdown: {
    nutrition: number | null;
    additives: number | null;
    certification: number | null;
  };
  reasons: Array<{
    title: string;
    desc: string;
    severity: "good" | "normal" | "caution" | "danger" | "info";
    source: string;
  }>;
  warnings: Array<{
    code: string;
    message: string;
    blocking: boolean;
    severity: "good" | "normal" | "caution" | "danger" | "info";
  }>;
  personalWarnings: Array<{
    code: string;
    message: string;
    blocking: boolean;
    severity: "good" | "normal" | "caution" | "danger" | "info";
  }>;
  computable: boolean;
};
```
