# 안전 감시 정책

마지막 업데이트: 2026-05-13

이 문서는 저장 제품, 최근 스캔 제품, 임시 제품을 로컬에서 재분류해 사용자가 즉시 확인해야 할 위험을 표시하는 기준을 정의한다.

## 대상 범위

안전 감시는 브라우저 로컬 데이터만 사용한다.

| 대상           | 저장 위치            | 감시 목적                                         |
| -------------- | -------------------- | ------------------------------------------------- |
| 저장 제품      | `kfs:saved-products` | 사용자가 계속 확인하려는 제품의 위험 재노출       |
| 최근 스캔 제품 | `kfs:recent`         | 최근 조회 제품의 회수·개인 기준 위험 확인         |
| 임시 제품      | `kfs:provisional`    | OCR·사용자 제보 제품의 저신뢰·검수 필요 상태 확인 |

동일 바코드가 여러 목록에 있으면 하나의 감시 항목으로 병합한다. 바코드가 없으면 제품 ID를 기준으로 병합한다.

## 감시 사유

| 사유                | 심각도 | 기준                                         |
| ------------------- | ------ | -------------------------------------------- |
| 회수·판매중지       | 긴급   | 제품에 회수 이력이 있음                      |
| 성분 알림 직접 일치 | 긴급   | 라벨 표기 성분과 선택한 성분 알림이 일치     |
| 고위험 첨가물       | 긴급   | 점수 정책에서 `high_risk_additive` 경고 발생 |
| 성분 알림 추정      | 주의   | 원재료에서 선택한 성분 알림이 감지됨         |
| 식이 선호 경고      | 주의   | 선택한 식이 선호와 맞지 않는 성분이 감지됨   |
| 저신뢰 데이터       | 주의   | AI·사용자 제보 제품 confidence가 0.7 미만    |
| 영양 정보 부족      | 정보   | 필수 영양성분 부족으로 점수 산출 불가        |
| 검수 필요           | 정보   | 제품 상태가 `needs_review`                   |

한 제품에 여러 사유가 있으면 가장 높은 심각도로 표시한다.

## 정렬 기준

감시 항목은 다음 순서로 정렬한다.

1. 심각도: 긴급, 주의, 정보
2. 사유 우선순위: 회수, 성분 알림 직접 일치, 고위험 첨가물, 성분 알림 추정, 식이 선호 경고, 저신뢰 데이터, 영양 정보 부족, 검수 필요
3. `updatedAt` 최신순

## 화면 표시

`/history` 화면은 기록 인사이트 아래에 안전 감시 패널을 표시한다.

- 감시 중인 전체 제품 수
- 회수 알림 수
- 긴급 알림 수
- 주의 알림 수
- 저장 제품에 포함된 알림 수
- 상위 3개 감시 항목과 제품 상세 링크

## 한계

- 서버 푸시 알림 기능이 아니다.
- 백그라운드 자동 재조회 기능이 아니다.
- 현재 로컬에 저장된 제품 스냅샷 기준으로만 계산한다.
- 회수 정보는 제품 조회 시점에 보강된 값만 감시한다.
